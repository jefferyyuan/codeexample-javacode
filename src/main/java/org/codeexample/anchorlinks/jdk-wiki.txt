<p>The <b>Java Development Kit</b> (<b>JDK</b>) is an implementation of either one of the <a href="/wiki/Java_SE" title="Java SE" class="mw-redirect">Java SE</a>, <a href="/wiki/Java_EE" title="Java EE" class="mw-redirect">Java EE</a> or <a href="/wiki/Java_ME" title="Java ME" class="mw-redirect">Java ME</a> platforms<sup id="cite_ref-1" class="reference"><a href="#cite_note-1"><span>[</span>1<span>]</span></a></sup> released by <a href="/wiki/Oracle_Corporation" title="Oracle Corporation">Oracle Corporation</a> in the form of a binary product aimed at <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a> developers on <a href="/wiki/Solaris_(operating_system)" title="Solaris (operating system)">Solaris</a>, <a href="/wiki/Linux" title="Linux">Linux</a>, <a href="/wiki/Mac_OS_X" title="Mac OS X" class="mw-redirect">Mac OS X</a> or <a href="/wiki/Windows" title="Windows" class="mw-redirect">Windows</a>.<sup id="cite_ref-2" class="reference"><a href="#cite_note-2"><span>[</span>2<span>]</span></a></sup> Since the introduction of the <a href="/wiki/Java_(software_platform)" title="Java (software platform)">Java</a> platform, it has been by far the most widely used Software Development Kit (<a href="/wiki/Software_development_kit" title="Software development kit">SDK</a>).<sup class="Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (January 2013)">citation needed</span></a></i>]</sup> On 17 November 2006, Sun announced that it would be released under the <a href="/wiki/GNU_General_Public_License" title="GNU General Public License">GNU General Public License</a> (GPL), thus making it <a href="/wiki/Free_software" title="Free software">free software</a>. This happened in large part on 8 May 2007, when Sun contributed the source code to the <a href="/wiki/OpenJDK" title="OpenJDK">OpenJDK</a>.<sup id="cite_ref-3" class="reference"><a href="#cite_note-3"><span>[</span>3<span>]</span></a></sup></p>

<h2><span class="mw-headline" id="JDK_contents">JDK contents</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Java_Development_Kit&amp;action=edit&amp;section=1" title="Edit section: JDK contents">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The JDK has as its primary components a collection of programming tools, including:</p>
<ul>
<li><a href="/wiki/AppletViewer" title="AppletViewer">appletviewer</a> – this tool can be used to run and debug Java applets without a web browser</li>
<li>apt – the <a href="/wiki/Metadata_facility_for_Java" title="Metadata facility for Java">annotation-processing tool</a><sup id="cite_ref-apt_4-0" class="reference"><a href="#cite_note-apt-4"><span>[</span>4<span>]</span></a></sup></li>
<li>extcheck – a utility which can detect JAR-file conflicts</li>
<li>idlj – the IDL-to-Java compiler. This utility generates Java <a href="/wiki/Language_binding" title="Language binding">bindings</a> from a given <a href="/wiki/Java_Interface_Definition_Language" title="Java Interface Definition Language">Java IDL</a> file.</li>
<li>jabswitch – the Java Access Bridge. Exposes assistive technologies on Microsoft Windows systems.</li>
<li>java – the <a href="/wiki/Loader_(computing)" title="Loader (computing)">loader</a> for Java applications. This tool is an interpreter and can interpret the class files generated by the <a href="/wiki/Javac" title="Javac">javac</a> compiler. Now a single launcher is used for both development and deployment. The old deployment launcher, jre, no longer<sup class="plainlinks noprint asof-tag update" style="display:none;"><a class="external text" href="//en.wikipedia.org/w/index.php?title=Java_Development_Kit&amp;action=edit">[update]</a></sup> comes with Sun JDK, and instead it has been replaced by this new java loader.</li>
<li><a href="/wiki/Javac" title="Javac">javac</a> – the <a href="/wiki/Java_compiler" title="Java compiler">Java compiler</a>, which converts source code into <a href="/wiki/Java_bytecode" title="Java bytecode">Java bytecode</a></li>
<li><a href="/wiki/Javadoc" title="Javadoc">javadoc</a> – the documentation generator, which automatically generates documentation from <a href="/wiki/Source_code" title="Source code">source code</a> comments</li>
<li>jar – the archiver, which packages related class <a href="/wiki/Library_(computer_science)" title="Library (computer science)" class="mw-redirect">libraries</a> into a single <a href="/wiki/Jar_(file_format)" title="Jar (file format)" class="mw-redirect">JAR file</a>. This tool also helps manage JAR files.</li>
<li>javafxpackager – tool to package and sign JavaFX applications</li>
<li>jarsigner – the jar signing and verification tool</li>
<li>javah – the C header and stub generator, used to write native methods</li>
<li>javap – the class file <a href="/wiki/Disassembler" title="Disassembler">disassembler</a></li>
<li>javaws – the <a href="/wiki/Java_Web_Start" title="Java Web Start">Java Web Start</a> launcher for JNLP applications</li>
<li><a href="/wiki/JConsole" title="JConsole">JConsole</a> – Java Monitoring and Management Console</li>
<li>jdb – the <a href="/wiki/Debugger" title="Debugger">debugger</a></li>
<li>jhat – Java Heap Analysis Tool (experimental)</li>
<li>jinfo – This utility gets configuration information from a running Java process or crash dump. (experimental)</li>
<li>jmap – This utility outputs the memory map for Java and can print shared object memory maps or heap memory details of a given process or core dump. (experimental)</li>
<li>jmc – Java Mission Control</li>
<li>jps – Java Virtual Machine Process Status Tool lists the instrumented HotSpot Java Virtual Machines (JVMs) on the target system. (experimental)</li>
<li>jrunscript – Java command-line <a href="/wiki/Shell_script" title="Shell script">script</a> <a href="/wiki/Shell_(computing)" title="Shell (computing)">shell</a>.</li>
<li>jstack – utility which prints Java <a href="/wiki/Stack_trace" title="Stack trace">stack traces</a> of Java threads (experimental)</li>
<li>jstat – <a href="/wiki/Java_Virtual_Machine" title="Java Virtual Machine" class="mw-redirect">Java Virtual Machine</a> statistics monitoring tool (experimental)</li>
<li>jstatd – jstat daemon (experimental)</li>
<li><a href="/wiki/Keystore" title="Keystore">keytool</a> – tool for manipulating the <a href="/wiki/Keystore" title="Keystore">keystore</a></li>
<li><a href="/wiki/Pack200" title="Pack200">pack200</a> – JAR compression tool</li>
<li>policytool – the policy creation and management tool, which can determine policy for a Java runtime, specifying which permissions are available for code from various sources</li>
<li><a href="/wiki/VisualVM" title="VisualVM">VisualVM</a> – visual tool integrating several <a href="/wiki/Command-line_interface" title="Command-line interface">command-line</a> JDK tools and lightweight<sup class="noprint Inline-Template" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Please_clarify" title="Wikipedia:Please clarify"><span title="The text near this tag may need clarification or removal of jargon. (October 2010)">clarification needed</span></a></i>]</sup> performance and memory <a href="/wiki/Profiling_(computer_programming)" title="Profiling (computer programming)">profiling</a> capabilities</li>
<li>wsimport – generates portable <a href="/wiki/JAX-WS" title="JAX-WS" class="mw-redirect">JAX-WS</a> artifacts for invoking a web service.</li>
<li>xjc – Part of the Java API for XML Binding (JAXB) API. It accepts an XML schema and generates Java classes.</li>
</ul>
<p>Experimental tools may not be available in future versions of the JDK.</p>
<p>The JDK also comes with a complete <a href="/wiki/Java_Runtime_Environment" title="Java Runtime Environment" class="mw-redirect">Java Runtime Environment</a>, usually called a <i>private</i> runtime, due to the fact that it is separated from the "regular" JRE and has extra contents. It consists of a <a href="/wiki/Java_Virtual_Machine" title="Java Virtual Machine" class="mw-redirect">Java Virtual Machine</a> and all of the class libraries present in the production environment, as well as additional libraries only useful to developers, such as the <a href="/wiki/Internationalization_and_localization" title="Internationalization and localization">internationalization</a> libraries and the <a href="/wiki/Interface_description_language" title="Interface description language">IDL</a> libraries.</p>
<p>Copies of the JDK also include a wide selection of example programs demonstrating the use of almost all portions of the <a href="/wiki/Java_API" title="Java API" class="mw-redirect">Java API</a>.</p>
<h2><span class="mw-headline" id="Ambiguity_between_a_JDK_and_an_SDK">Ambiguity between a JDK and an SDK</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Java_Development_Kit&amp;action=edit&amp;section=2" title="Edit section: Ambiguity between a JDK and an SDK">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The JDK forms an extended subset of a <a href="/wiki/Software_development_kit" title="Software development kit">software development kit</a> (SDK). It includes "tools for developing, debugging, and monitoring Java applications".<sup id="cite_ref-5" class="reference"><a href="#cite_note-5"><span>[</span>5<span>]</span></a></sup> Oracle strongly suggests that they now use the term "JDK" to refer to the Java SE Development Kit. The Java EE SDK is available with or without the "JDK", by which they specifically mean the Java SE 7 JDK.<sup id="cite_ref-6" class="reference"><a href="#cite_note-6"><span>[</span>6<span>]</span></a></sup></p>
<h2><span class="mw-headline" id="Other_JDKs">Other JDKs</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Java_Development_Kit&amp;action=edit&amp;section=3" title="Edit section: Other JDKs">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>In addition to the most widely used JDK discussed in this article, there are other JDKs commonly available for a variety of platforms, some of which started from the Sun JDK source and some which did not. All of them adhere to the basic Java specifications, but they often differ in explicitly unspecified areas, such as garbage collection, compilation strategies, and optimization techniques. They include:</p>
<p>In development or in maintenance mode:</p>
<ul>
<li><a href="/wiki/OpenJDK" title="OpenJDK">OpenJDK</a> / <a href="/wiki/IcedTea" title="IcedTea">IcedTea</a>;</li>
<li><a href="/wiki/GNU" title="GNU">GNU</a>'s <a href="/wiki/GNU_Classpath" title="GNU Classpath">Classpath</a> and <b><a href="/wiki/GCJ" title="GCJ" class="mw-redirect">GCJ</a></b> (The <a href="/wiki/GNU_Compiler_for_Java" title="GNU Compiler for Java">GNU Compiler for Java</a>);</li>
<li><a href="/wiki/Aicas" title="Aicas">Aicas</a> <a href="/wiki/JamaicaVM" title="JamaicaVM">JamaicaVM</a>;</li>
